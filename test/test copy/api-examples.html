computed和watch ​

01. computed计算属性 ​

  • 需要引入computedAPI来定义计算属性，并返回一个ref对象

    typescript
    // <script setup lang="ts">

import { computed } from 'vue'

const firstName = ref<string>('张')
const lastName = ref<string>('三三')


// 1. 直接定义：传入一个 getter 函数
const fullName1 = computed<string>(() => {
    return firstName.value + lastName.value
})

// 2. 定义 setter/getter 函数，传入一个对象
const fullName2 = computed<string>({
    get() {
        return firstName.value + lastName.value
    },
    set(value) {
        firstName.value = value.split('-')[0]
        lastName.value = value.split('-')[1]
    }
})

// 3. 访问 computed 定义的属性， 同 ref 的访问，需要使用 .value
console.log(fullName1.value)
console.log(fullName2.value)

02. watch监听器 ​

（1）watch ​

  • 需要引入watchAPI来定义一个监听器

  • watch()函数可以接受三个参数

    • 第一个参数：需要监听的变量

      • 监听ref | reactive定义的变量，可以直接使用变量名
      • 监听reactive对象中的某个属性，需要使用回调函数的形式返回属性名
      • 可以同时监听多个数据，需要使用数组来传入变量

    • 第二个参数：执行的回调函数

      • 回调函数有两个参数
        • 第一个参数是：变化后的新值，或者新值数组
        • 第二个参数是：变化前的旧值，或者旧值数组
        • 注意：如果监听的是对象，是无法感知这个变化的

    • 第三个参数：扩展配置对象

      • 可以配置immediate为true, 来进行初始立即监听
      • 可以配置deep为true, 来对对象进行深度监听
    typescript
    // <script setup>

import { ref, reactive, watch } from 'vue';

// 1. 监听 ref
const num = ref(10)
watch(
    num,
    (newVal, oldVal) => {
        console.log(newVal)
        console.log(oldVal)
    },
    { immediate: true, deep: true }
)

// 2. 监听 整个reactive对象， 此时会自动启用 深度监听
const state = reactive({ 
    name: 'vue', 
    age: 10 
})
watch(state, (newVal) => {
    console.log(newVal)
})

// 3. 监听 reactive 对象的【某个属性】，需要使用【回调函数】的形式
watch(() => state.age, (newVal, oldVal) => {
    console.log(newVal)
    console.log(oldVal)
})

// 4. 同时监听多个变量
watch([() => state.name, () => state.age], ([newName, newAge], [oldName, oldAge]) => {
    console.log(newName);
    console.log(newAge);

    console.log(oldName);
    console.log(oldAge);

})

（2）watchEffect ​

  • 需要引入watchEffectAPI

    • watchEffect不需要传参，它会默认立即执行一次，并在执行过程中，会自动收集副作用函数内部的响应式数据
    • 即，只要在函数内部使用了的响应式数据，那么watchEffect就会监听这些数据的变化

  • watchEffect只需要两个参数

    • 第一个参数：执行的回调函数或者说是副作用函数

    • 第二个参数：配置调用时机flush

      • pre：默认值，会在元素挂载或者更新之前执行——即提前执行
      • post：在元素挂载或者更新完成之后执行，同watchPostEffect
      • sync：强制同步触发，同watchSyncEffect
    js
    import {  watchEffect, watchPostEffect, watchSyncEffect  } from 'vue';

// watchEffect
watchEffect(()=>{
    console.log(state.name)
})
// 同
watchEffect(
    ()=>{
        console.log(state.name)
    },
    { flush: 'pre' }
)

// watchPostEffect
watchPostEffect(()=>{
    console.log(state.name)
})
// 同
watchEffect(
    ()=>{
        console.log(state.name)
    },
    { flush: 'post' }
)


// watchSyncEffect
watchSyncEffect(()=>{
    console.log(state.name)
})
// 同
watchEffect(
    ()=>{
        console.log(state.name)
    },
    { flush: 'sync' }
)

（3）对比 ​

  • watch需要传入具体的监听参数，watchEffect 不需要传入监听参数

  • watch 只有监听的属性变化时才执行，watchEffect 第一次会立即执行（以收集监听数据）

  • watch 可以直接监听 ref 和 reactive 绑定的对象，watchEffect 不可以（ref的值要.value，reactive的值要具体到内部属性）

  • watch 和watchEffect都无法监听未被绑定的属性

（4）停止监听🛑 ​

  • 通过调用watch函数或者watchEffect函数的返回值，可以停止监听

    js
    import { watch } from 'vue';

// 设置 watch 函数的返回值
const stop = watch(state, () => {
    // 调用 watch 的返回值 停止监听
    stop()
})
