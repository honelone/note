00. 前情提要 ​

（1）扁平化依赖 ​

  • 如果项目的依赖项层级过深，npm、yarn都会通过扁平化依赖解决问题

    • 即，项目的依赖，依赖的依赖，都会被拍平安装到项目的node_modules中

  • 项目的声明的依赖dependencies和devDependencies都会被安装

    • 但是依赖的依赖只有dependencies会被安装

  • 扁平化依赖虽然没解决了很多问题，但是会衍生出以下问题：

    • 扁平化依赖算法复杂，需要消耗较多的性能
    • 因为所以依赖都被安装到项目中，会带来依赖非法访问的问题
      • 即项目中没有定义的依赖，在依赖的依赖定义了，导致无需声明就可以引用
      • 这种情况就是幽灵依赖

01. npm ​

（1）简介 ​

  • 安装npm包分为两种情况

    • 全局安装：npm install XXX -g
      • 会在node安装目录生成一个node_modules 文件夹
    • 局部安装：npm install XXX
      • 会在当前目录下生成一个 node_modules 文件夹

  • --save或者-S：会添加到dependencies中。生成环境需要用到

  • --save-dev或者-D：会添加到devDependencies中，开发环境需要用到

  • -g：将依赖安装到全局

（2）使用 ​

  • 安装：install，或者简写为i

    shell
    # 安装当前项目中所有 node 包
npm insatll # npm i

# 安装指定 node 包
npm install XXX # npm i XXX

02. cnpm ​

（1）简介 ​

  • cnpm：有时候不方便下载一些node包，可以使用cnpm设置镜像，方便我们下载node包

  • 介绍：

    • cnpm的镜像仓库在国内，所以速度会比npm快很多
    • cnpm不会产生package-lock.json文件
      • 所以它安装时也是不会去读取lock文件中的缓存的

（2）使用 ​

  • 安装cnpm：

    shell
    # 安装 cnpm
npm install -g cnpm --registry=https://registry.npm.taobao.org

# 设置 cnpm 下载镜像
cnpm config get registry # https://r.npm.taobao.org/

# 查看版本信息
cnpm -v

03. npx ​

  • npx是npm5.2之后自带的一个命令
    • 最常见的是，用npx来调用项目中的某个模块的命令
    • 即，它会到当前目录下的node_modules中去寻找对应的命令
    • 如果没有。才会去全局下寻找

04. yarn ​

（1）简介 ​

  • yarn：是另一个node包管理工具

    • 早期的npm存在很多缺陷，安装依赖速度慢、版本依赖混乱等
    • yarn是为了弥补早起npm的一些缺陷而出现的

  • 介绍

    • 网络性能更好
    • 采用缓存机制，实现了离线模式，当然现在npm也有

  • yarn也有lock文件用来锁定版本号

  • 安装速度

    • yarn的安装速度要比npm快
    • 这是因为yarn采用了并行安装依赖的方式进行安装
    • 而npm采用的是串行的方式

（2）使用 ​

  • 安装yarn

    shell
    # 安装 yarn
npm install --global yarn

# 查看版本信息
yarn --version

05. pnpm ​

（1）简介 ​

  • pnpm：可以理解成performant npm的缩写
  • 优点：
    • 快速：pnpm比其他包管理器快 2 倍
    • 支持monorepos：pnpm内置了对存储库中的多个包的支持
    • 高效：node_modules中的文件链接自特定的内容寻址存储库
    • 严格：pnpm默认创建了一个非平铺的node_modules，因此代码无法访问任意包
  • pnpm做了什么？
    • 当我们使用npm或者yarn时
    • 如果我们有多个项目都依赖相同的依赖包
    • 那么就会存多份相同的依赖包
  • 而如果使用pnpm，依赖包会被存放在一个统一的位置
    • 如果你对同一依赖包使用相同的版本，那么磁盘上只有这个依赖包的一份文件
    • 如果你对同一依赖包需要使用不同的版本，则仅有版本之间不同的文件会被存储起来
    • 所有文件都保存在硬盘上的统一的位置：
      • 当安装软件包时， 其包含的所有文件都会硬链接到此位置，而不会占用 额外的硬盘空间
      • 这让你可以在项目之间方便地共享相同版本的 依赖包；

（2）使用 ​

  • 安装pnpm：

    shell
    # 安装 pnpm
npm install -g pnpm
