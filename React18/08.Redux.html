Redux ​

Redux是一个帮助我们管理state的状态管理器

01. 三大核心理念和原则 ​

  • 三个核心理念

    • Store：用于存储数据、状态
      • Store的数据来自于Reducer返回的state
      • 数据更新时，通过store.dispatch()派发给Reducer，由Reducer生成新的state返回
      • 最后更新到Store
    • Action：用来更新Store中数据的，且必须以这种方式更新
      • 需要通过dispatch派发更新
    • Reducer：一个纯函数，用于将纯如的state和action结合起来生成一个新的state返回

    纯函数，即

    • 相同的输入一定是相同的输出
    • 不依赖于外部状态
    • 不产生副作用，比如修改全局变量，修改参数，改变外部的存储变量等操作

    纯函数的作用：

    • 只需要关心业务逻辑，不需要关心传入的内容、依赖的变量

  • 三个原则

    • 单一数据源
      • 整个项目的State被存储在一个Object Tree中，并且这个对象只存储在一个Store中
      • 虽然可以创建多个Store，但这样不利于数据的维护
      • 单一数据源可以让项目的state变得方便维护、追踪、修改
    • state是只读的
      • 修改state的方法只能是action派发的事件，不能在其它任何地方去手动修改state
      • 保证所有修改都被集中化处理
    • 使用纯函数来执行修改
      • 通过reducer将state和action联系起来，并返回一个新的state
      • 可以将一个大的reducer拆分为多个小的reducer，用于操作不同的state tree部分
      • 所有reducer都应该是纯函数

            return state
    }

    updateName(name) {
      dispatch(updatNameAction(name));
    },
  };
};

05. ReduxToolKit ​

（1）介绍 ​

  • ReduxToolKit即RTK，本质上是对Redux做的一层封装，让Redux的使用更加简洁

  • 核心 API

    • configureStore：用于替代原来的createStore

      • 包装createStore，提供了简化的配置选项和良好的默认值
      • 可以自动组合reducers
      • 添加Redux中间件
      • 默认包含redux-thunk
      • 默认启用Redux DevTools Extension

    • createSlice：用于切片reducer

      • 接收一个reducer函数的对象、切片名称和初始状态值，并自动生成切片reducer
      • 带有相应的actions

    • createAsyncThunk：

      • 接受一个动作类型字符串，和一个返回Promise的函数
      • 并生成一个Pending | Fulfilled | Rejected

（2）示例 ​

  • 代码示例

07. connect原理 ​

jsx
import { PureComponent } from 'react';
import store from './store';

export default function connect(mapStateToProps, mapDispatchToProps) {
  // 返回的是一个高阶组件
  return (OriginComponent) => {
    class NewComponent extends PureComponent {
      constructor(props) {
        super(props);

        this.state = mapStateToProps(store.getState());
      }
      componentDidMount() {
        // 订阅数据变化
        this.unsubscribe = store.subscribe(() => {
          // this.forceUpdate()
          this.setState(mapStateToProps(store.getState()));
        });
      }
      componentWillUnmount() {
        this.unsubscribe();
      }
      render() {
        const stateObj = mapStateToProps(store.getState());
        const dispatchObj = mapDispatchToProps(store.dispatch);

        return <OriginComponent {...this.props} {...stateObj} {...dispatchObj} />;
      }
    }

    return NewComponent;
  };
}

  • redux-thunk可以让dispatch处理对象和函数

    jsx
    function thunk(store) {
  const next = store.dispatch;

  function dispatchThunk(action) {
    if (typeof action === 'function') {
      action(store.dispatch, store.getState());
    } else {
      next(action);
    }
  }

  store.dispatch = dispatchThunk;
}
