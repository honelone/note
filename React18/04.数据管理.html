01. state和setState ​

（1）state和props ​

  • 在类组件中

    • 需要将当前组件的数据挂载到this.state上
    • 当前组件接收的父组件的数据会挂载到this.props上
    • 挂载操作需要在constructor中进行
    jsx
    class App extends PureComponent {
    // 如果什么也不做，可以不写 constructor
    render() {
        return null
    }
}
class App extends PureComponent {
    // 如果需要初始化数据，则需要在 consturctor 中进行
    constructor() {
        // 如果有 constructor ,则必须执行 super()
        super()
        // state 需要在 constructor 中进行初始化
        this.state = {
            message: 'Hello, Ruovan'
        }
    }
    render() {
        return <div>{ this.state.message }</div>
    }
}

class App extends PureComponent {
    // 如果需要接收 props,有三种方式
    // 1. 不写 constructor， 内部会默认执行
    // 2. 写 constructor ，并将 props 通过 super() 传递给父类， 父类会绑定到 this.props 上
    constructor(props) {
        super(props)
    }
    // 3. 写 constructor, 并手动将其挂载到 this.props 上
    constructor(props) {
        super()
        this.props = props
    }
    render() {
        return <div>{ this.props.message }</div>
    }
}

（2）setState ​

  • 在类组件中，如果需要对state进行修改，只能通过setState

  • 这是因为React中没有对数据进行劫持，需要setState才能让React感知到，从而重新渲染组件

  • 特别注意，不能通过其它方式去修改state，修改的时候可以先拷贝一份state，修改后再通过setState赋值回去

  • setState是一个异步操作，即执行完setState后并不能立即获取到最新的state结果

    • 异步操作可以有效提升性能，即将当前队列上的所有更新获取到，然后在下一次队列批量更新
    • 可以避免render()函数被频繁调用
    • 如果同步更新state，但还没有执行render函数，那么就会导致state和props不能保持同步，这样会导致更多问题产生

  • setState可以接受两个参数

    • 第一个参数：用于更新state数据
    • 第二个参数：回调函数，在数据更新后执行
    jsx
    function changeState() {
  this.setState(
    {
      message: 'Hello, React!',
    },
    // 可选参数，可以将之前的 state | props 传进来
    (state, props) => {
      // 可以在这里重新对 state 进行逻辑处理
      console.log('update: ', this.state.message);
    },
  );
}

// 或者，也可以在 componentDidUpdate 生命周期函数内 获取到这个更新后的数据

（3）setState的异步和同步 ​

  • 在React18之前

    • 在组件的生命周期，或者React的合成事件中，setState是异步操作
    • 在setTimeout，或者原生 DOM 事件中，setState是同步操作

  • 但在React18中

    • 默认的所有的操作都被放到了批处理（异步处理）中

    • 而如果要在同步代码中拿到，则需要另一个函数flushSync辅助

      jsx
      import { flushSync } from 'react-dom';

flushSync(() => {
  this.setState({
    message: 'Hello, World!',
  });
});

// 这里可以同步拿到
console.log(this.state.message);

02. Context ​

（1）介绍 ​

  • React提供了一个 API：Context
    • 用于在组件之间共享数据，而不必通过组件树的逐层传递props
    • 主要用于全局数据的共享

（2）使用 ​

  • 通过React.createContext()来创建一个Context全局共享的对象

    • 可以接收一个对象参数，用作默认数据
    • Context对象
      • 提供一个Provider组件
        • 用于向包裹的组件及其后代组件，提供需要共享的数据
        • 使用value属性来提供这些数据
        • 可以嵌套使用，内部会覆盖外层数据
      • 提供了一个Consumer组件
        • 用于订阅Context共享的数据
        • 内部是一个函数，会接收当前Context的值
        • 当订阅的数据发生变化，会重新渲染内部组件
        • 可以在一个组件中使用不同的Context对象
        • 主要用于函数式组件，或者需要使用多个Context时

（3）示例 ​

  • 创建Context

    jsx
    // my-context.js
export default const MyContext = React.createContext()

03. 获取 DOM ​

（1）获取方式 ​

  • 可以通过refs来获取 DOM 元素，主要有三种方式

    • 传入字符串：通过this.refs.传入的字符串格式获取对应的元素
    • 传入一个对象：通过 React.createRef() 方式创建一个ref对象绑定到元素上
    • 传入一个函数：将函数绑定到元素上，函数会在 DOM 挂载时进行回调，这个时候将获取到的元素对象保存起来
    jsx
    class App extends PureComponent {
  constructor() {
    super();

    this.worldRef = creatrRef();
    this.reactRef = null;
  }
  getDom() {
    // 1. 在元素上绑定一个 ref 字符串
    console.log(this.refs.helloRef);

    // 2. 通过 React.createRef() 创建一个 ref 对象，然后绑定到元素上
    // -- 获取到的元素绑定在这个对象上的 current 属性上
    console.log(this.worldRef.current);

    // 3. 通过回调函数的形式保存元素对象
    console.log(this.reactRef);
  }

  render() {
    return (
      <div>
        <div ref="helloRef">Hello</div>
        <div ref={this.worldRef}>World</div>
        {/* 这个 e 就是元素对象，这个函数会在DOM挂载时自动执行，然后就可以获取到 元素对象 */}
        <div ref={(e) => (this.reactRef = e)}>React</div>
      </div>
    );
  }
}

（2）ref 类型 ​

  • ref的值会根据节点类型的不同而不同

    • 当ref用于HTML元素时，会接收底层的 DOM 元素作为其ref对象的current属性
    • 当ref用于自定义类组件时，ref对象接收组件的挂载实例作为其current属性
    • ref不能用于函数式组件，因为它们没有实例

  • 如果要获取函数式组件的元素，我们可以通过React.forwardRef——ref的转发

    • 同样的，需要用forwardRef将函数式组件包裹起来
    • 即，实际上是将ref绑定在我们想要获取的元素上，而不是实例上（没有实例）
    jsx
    import { forwardRef } from 'react'

// ref 将作为第二个参数传递进来，然后我们将其 绑定在我们想要获取的元素上
const Child = forwardRef((props, ref) => {
    return (
    	<div>
            { /* 父组件获取的实际上是这个DOM元素 */ }
        	<div ref={ref}>Hello</div>
        </div>
    )
})

export default Child


# 父组件
class Father extends PureComponent {

04. 性能优化SCU（暂存，要改） ​

（1）React 的更新机制 ​

  • 渲染流程：JSX —— 虚拟DOM —— 真实DOM
  • 更新流程：
    • props | state 改变
    • render函数重新执行
    • 产生新的 DOM 树
    • 新旧 DOM 树进行Diff
    • 计算出差异进行更新
    • 更新到真实 DOM 树

（2）Diff ​

  • React的Diff算法（O(n)）

    • 同层节点支架相互比较，不会跨节点比较
    • 不同类型的节点，产生不同的树结构
    • 开发中，可以通过key来指定哪些节点在不同的渲染下保持稳定

  • key的优化

    • key必须是唯一的
    • key不要使用随机数，每次都会生成不一样的数字
    • key不能用index，对性能没有优化

（3）SCU ​

  • 调用render时，应该是组件依赖的数据props | state发生改变时，再调用自己的render方法

    • 但实际上却是只要数据发生变化，所有组件都会重新渲染（调用render）

  • 因此，可以使用shoulComponentUpdate生命周期函数，来指定是否调用当前组件的render方法

    • 这种性能优化方法就是SCU

  • SCU可以接受参数，并且必须要有返回值

    • 两个参数：
      • 参数一：nextProps修改之后，最新的props属性
      • 参数二：nextState修改之后，最新的state属性
    • 返回值：
      • 返回值是一个boolen类型，true为调用render，false为不调用render
      • 默认的返回为true

  • SCU的作用，通过判断props | state中的数据是否发生改变，来返回true | false

    jsx
    shouldComponentUpdate(nextProps, nextState) {
    if(this.props.message !== nextProps.message) {
        return true
    }else {
        return false
    }
}

（4）PureComponent ​

  • SCU需要在每个类组件上都主动添加去实现，比较繁琐

  • 实际上，React已经默认帮我们实现好了，就是这个PureComponent类

    • 其内部有一个shallowEqual(oldValue, newValue)方法
    • 这个方法用于进行两个数据的浅层对比，从而实现上面的SCU的功能
    jsx
    // 即，将原来的 Component 替换为 PureComponent， react 内部会自动进行判断比较
import { PureComponent } from 'react';

class App extends PureComponent {}

（6）不可变数据 ​

  • 对于this.state中的数据，永远不要直接去修改它，否则，可能导致shallowEqual的对比错误

    • 一般指的是引用类型的数据
    jsx
    class App extends PureComponent {
  constructor() {
    super();
    this.state = {
      arr: [1, 2, 3, 4],
    };
  }
  handleUpdate() {
    // 这是错误的做法，因为直接修改了 this.state
    // -- 会导致 shallowEqual 判断为数据未发生改变，引用类型的数据
    this.state.arr.push(5);
    this.setState({
      arr: this.state.arr,
    });

    // 需要先断开与原数据的引用, 复制一份新的数据，更新之后再 setState
    const newArr = [...this.state];
    newArr.push(5);
    this.setState({
      arr: newArr,
    });
  }
}
