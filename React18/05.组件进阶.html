01. 高阶组件 ​

（1）介绍 ​

  • 高阶函数：（至少满足以下条件之一）

    • 接收一个或多个函数作为参数
    • 输出一个函数
    • 如：Array.filter()、Array.map()等

  • 高阶组件：Higher Order Componnet，简称HOC

    • 是一个函数
    • 接收一个组件作为参数
    • 并返回一个新组件

  • 代码示例

    jsx
    const NewComponent = higherOrderComponnet(OldComponent);

function higherOrderComponnet(OldComponent) {
  class NewComponent extends PureComponent {
    render() {
      // 将传入的 组件参数 返回
      return <OldComponent />;
    }
  }
  // 可以通过 displayName 属性来修改组件的名称
  NewComponent.displayName = 'MyComponent';
  // 返回一个新的组件
  return NewComponent;
}

  • 高阶组件实际上是

    • 对传入的组件做了一层拦截，进行了自定义处理
    • 通常用于可以复用逻辑的多个组件，将其逻辑抽离，放在高阶组件中处理

  • 高阶组件并不是React API的一部分，它是基于React的组合特性而形成的设计模式

（2）应用 ​

  • 对props进行增强

    • 将多个需要注入某些props的组件，通过高阶组件进行拦截
    • 在高阶组件提供这些数据，并将其挂载到传入的组件参数上
    jsx
    // 定义一个高阶组件
function enhancedComponent(OriginComponent) {
  class NewComponent extends PureComponent {
    constructor() {
      super();
      this.state = {
        message: '高阶组件注入数据',
      };
    }
    render() {
      {
        /* 这里的 this.state 是当前组件的 数据； 这里的 this.props 是返回的新组件上的属性，需要将其注入到原组件上 */
      }
      return <OriginComponent {...this.state} {...this.props} />;
    }
  }

  return NewComponent;
}

// 原函数式组件
const OtherComponent = enhancedComponent((props) => {
  return <h2>传入高阶组件的组件，{props.message}</h2>;
});

// 原类组件
class App extends PureComponent {}
export default enhancedComponent(App);

（3）意义 ​

  • 复用组件逻辑、简化了代码

  • 大量使用HOC，会导致难以调试，因为嵌套了大量组件，还可能导致props冲突

  • 更好的方法是Hooks

02. (非)受控组件 ​

（1）受控组件 ​

  • 对于表单元素，如果绑定了value属性，那么这个元素就会变成受控组件

    • 如：<input>、<textarea>、select

  • 这是因为，在React中是没有双向绑定这个概念的

    • 所以，在绑定了value属性的同时，还需要绑定onChange事件
    • 并在onChange事件中主动去更新绑定的value

  • 因此，在受控组件中，表单的数据是由React组件来管理的

    jsx
    class App extends PureComponent {
  constructor() {
    super();

    this.state = {
      name: '',
      password: '',
      isCheck: false, // 单个 checkbox
      // 多个 checkbox，需要通过数组对象来控制，每个对象对应一个 checkbox ，并单独有自己的 isChecked 属性
      hobbies: [
        {
          value: '',
          text: '',
          isChecked: '',
        },
      ],
      fruit: '',
    };
  }
  // 在一个方法里处理多个 input
  handleInputChange(event) {
    this.setState({
      // 获取到元素上绑定的 name 属性， 通过 动态属性名 来设置不同的值
      [event.target.name]: event.target.value,
    });
  }
  // 处理单个 checkbox
  handleCheckboxChange(event) {
    this.setState({
      isCheck: event.target.checked,
    });
  }
  // 处理多个 checkbox
  handleHobbiesChange(event, index) {
    const hobbies = [...this.state.hobbies];
    hobbies[index].isChecked = event.target.checked;
    this.setState({
      hobbies,
    });
  }
  // 多选
  handleSelectChange(event) {
    // 可以从元素上获取到所有已选择的选项： event.target.selectedOptions
    const options = Array.from(vent.target.selectedOptions);
    const values = options.map((item) => item.value);
    const values = options.map(
      (item) => item.value,
      (item) => item.value,
    );
    this.setState({ fruit: values });
  }

  render() {
    const { name, password, isCheck, hobbies, fruits } = this.state;
    return (
      <div>
        <input name="name" value={name} onChange={(event) => this.handleInputChange(event)} />
        <input
          name="password"
          value={password}
          onChange={(event) => this.handleInputChange(event)}
        />

        {/* 需要注意， type=checkbox 需要绑定 checked , 同样取值的时候也是 .checked */}
        <input
          type="checkbox"
          name="check"
          checked={isCheck}
          onChange={(event) => this.handleCheckboxChange(event)}
        />

        {/* 对于 checkbox 组需要通过数组对象来控制 */}
        <div>
          {hobbies.map((item, index) => {
            return (
              <input
                type="checkbox"
                chekced={item.isChecked}
                onChange={(e) => this.handleHobbiesChange(e, index)}
              />
            );
          })}
        </div>

        {/* 单选 | 多选 需要设置 multiple */}
        <select value={fruit} onChange={(e) => this.handleSelectChange(e)}></select>
      </div>
    );
  }
}

（2）非受控组件 ​

  • 将表单元素交给 DOM 节点来处理
    • 即，使用ref来获取元素，然后获取元素上的值，并设置到state上
  • 一般不建议这种方式，因为操作了 DOM 元素
jsx
class App extends PureComponent {
  constructor() {
    super();

    this.state = {
      inputValue: '',
    };
    this.inputRef = createRef();
  }
  componentDidMount() {
    // 需要使用 原生的监听方法 去实时监听 表单元素值 的变化
    this.inputRef.current.addEventListener();
  }

  render() {
    return (
      <div>
        {/* 可以通过 defaultValue 来设置默认值， 如果使用 value 设置，则会变成 受控组件 */}
        <input defaultValue={this.state.inputValue} ref={this.inputRef} />
      </div>
    );
  }
}

03. 其它组件 ​

（a）portals ​

  • Portal：即传送门，类似于Vue3的teleport，可以将子节点渲染到任意DOM节点

    • 使用ReactDOM.createPortal(child, container)
      • 第一个参数：渲染的子元素
      • 第二个参数：需要挂载的目标 DOM 元素
    jsx
    import { createPortal } from 'react-dom'

class Father extends PureComponent {
	render() {
        return (
        	<div>
                { /* 默认子节点在这里 */ }
            	<Child />
                { /* 使用 Portals 可以将其挂载到其它节点上 */ }
                {
                    createPortal(<Child />, document.querySelector('#app'))
                }
            </div>
        )
}

（b）<Fragment> ​

  • Fragment：即文档碎片，类似于Vue2的template，可以用于包裹多个子元素，但不会渲染fragment标签

    • 使用<></>空标签，或者<Fragment></Fragment>包裹子元素
    • 如果需要提供key，则不能使用空标签
    • 还是建议使用Fragment，方便查看
    jsx
    function App() {
  return (
    <Fragment>
      <div>1</div>
      <div>2</div>
    </Fragment>
  );
}

（c） <StrictMode> ​
  • 使用React.StrictMode组件来开启内部子元素的严格模式检查，包括所有后代元素
  • 主要用于：
    • 识别不安全的生命周期
    • 检测过时的ref API
    • 检测废弃的findDOMNode方法
    • 检测过时的context API
    • 检查意外的副作用
      • 组件的constructor会调用两次
