Nuxt3 常用的内置 API ​

01. 逻辑复用composables ​

（1）自动引入 ​

  • 在 composables 目录下模块的同名导出会被自动引入

    • 例如，composables 下有个 useFoo 模块
    typescript
    // composables/useFoo.ts
export const useFoo = 100;

// pages/
// 在各个页面中，这个模块 useFoo 将被按需自动引入
console.log(useFoo);

    • 例如，composables/useDark 下有个 index 模块
    typescript

export const useDark = 'yes';

// pages/
// useDark 将被按需自动引入

02. 状态共享useState ​

（1）基本概念 ​

  • 使用 useState 可以定义简单的共享状态

    • useState 的第一参数为唯一 key，第二参数为初始化的工厂函数
    • useState 只允许在生命周期中使用
    typescript
    useState<T>(key: string, init?: () => T): Ref<T>
    1

（2）全局共享 ​

  • 例如在 composables 中定义 useCounter

    typescript
    // composables/useCounter.ts
// composables 下模块的同名导出将被自动按需引入
export const useCounter = () => useState('counter', () => 100);
    1
    2
    3
    • 此时在其它页面使用 useCounter，所有页面的状态是共享的
    html
    <!-- pages/index.vue -->
<script lang="ts" setup>
  const conuter = useCounter(); // 这是按需自动引入的
</script>

<template>
  <div>
    <!-- 顶层的 div 包裹是必需的，不然跳转会丢失状态 -->
    <div @click="conuter++">{{ conuter }}</div>
    <NuxtLink to="/home">to home</NuxtLink>
  </div>
</template>

03. 缓存useCookie ​

  • 使用useCookie来缓存一些信息到cookie中，可以接收两个参数

    • 第一个参数：用于设置Cookie值的名字
    • 第二个参数：可选择配置项
      • 选项值同普通的Cookie设置项

  • 例

    typescript
    const counter = useCookie('counter', {
  maxAge: 3600,
  htttpOnly: true,
  secure: true,
});

04. useHead ​

（1）搜索引擎优化 ​

  • Nuxt3对搜索引擎SEO友好，可以通过useHead和useMeta来设置页面的title和meta

  • 例

    html
    <template>
  <div class="">About</div>
</template>
<script setup>
  useHead({
    title: '这里设置的 title 用于设置网站标题，会显示在页签上',
    meta: [
      { name: 'description', content: '这里是一段关于当前页面的描述' },
      { name: 'keywords', content: '这里设置当前页面的关键词，多个关键词用`,`分隔' },
    ],
  });
</script>

        crossorigin: '',

（3）definePageMeta ​

  • 另外，可以对每个页面单独使用definePageMeta来设置头信息

    typescript
    // pages/about.vue
definePageMeta({
  title: 'About Page',
});

// layouts/default.vue
useHead({
  meta: [
    {
      name: 'og:title',
    },
  ],
});
