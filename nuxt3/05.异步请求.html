Nuxt3 的请求 ​

01. 数据获取 ​

  • nuxt3 中内置了以下四种请求的方法
    • useFetch、useLazyFetch、useAsyncData、useLazyAsyncData

（1）useAsyncData ​

typescript
const { data, pending, refresh, error } = useAsyncData(key, handler, options?)
1

  • 三个参数：

    • key：一个唯一的键，以确保可以跨请求正确地对数据提取进行重复数据删除

      • 如果您不提供密钥，则会useAsyncData为您生成一个对实例的文件名和行号唯一的密钥。

    • handler : 一个返回值的异步函数

    • 选项options，可选项

      typescript
      interface Options {
  lazy: boolean; // 是否在加载路由后才请求该异步方法，默认为 false
  default: Function; // 在异步函数解析之前设置 数据默认值 的工厂函数，结合lazy:true选项特别有用
  server: boolean; // 是否获取服务器上的数据，默认为 true
  transform: Function; // 可用于`handler`在解析后更改 函数结果 的返回函数
  pick: Array<string>; // 函数结果 只会返回这个数组中指定的 key
  watch: any; // 观察反应源以自动刷新
  initialCache: boolean; // 是否跳过有效负载缓存以进行初始提取， 默认为 true
  immediate: boolean; // 是否阻止请求立即触发， 默认为true
}

  • 返回结果

    • data ：返回的异步函数的结果
    • pending ：是否在请求状态中
    • refresh / execute：一个函数，可以用来刷新handler函数返回的数据
    • error：数据获取失败时的错误对象

  • 基本使用方法

    typescript
    // 返回一个 Promise
const { data, error } = await useAsyncData('count', () => {
  return new Promise((resolve) => resolve('mock'));
});

// 请求数据，使用内置的  `$fetch` 可以直接进行请求
const { data, error } = await useAsyncData('count', () => {
  return $fetch('http://...'); // 请求地址
});

（2）useFetch ​

typescript
const { data, pending, refresh, error } = useFetch(url, options?)
1

  • useFetch实际上是对useAsyncData 和$fetch的封装

    • 它会根据URL和fetch参数自动生成一个key，同时推断出API的响应类型
    • useFetch 会在内部自动使用 $fetch 作为请求方法，所以没有 useAsyncData 灵活

  • 基本使用方法

    typescript
    const { data, error } = await useFetch('http://...'); // 你的请求地址
    1

（3）Lazy ​

  • useFetch 和 useAsyncData 会阻塞导航的跳转

    • 例如，index 页面跳转到 home 页面
    • 如果 home 页面有 useFetch 请求，则会会阻塞导航的跳转
    html
    <!-- pages/home.vue -->
<script lang="ts" setup>
  // 以下请求将会阻塞导航的跳转
  const { data } = await useFetch('http://localhost:4000');
</script>

<template>
  <div>{{ data }}</div>
</template>

03. 路由中间件middleware ​

（1）基本概念 ​

  • 路由中间件的作用类似于VueRouter4中的拦截器，可以通过中间件来设置路由守卫

    • 可以在文件名之后加.global，表示用于全局的路由守卫，如default.global.ts

    • 如果不加.global，则需要到每个页面单独设置，通过definePageMeta设置

      html
      <script setup>
  definePageMeta({
    middleware: ['default'],
  });
</script>
      1
      2
      3
      4
      5

（2）定义 ​

  • 使用defineNuxtRouteMiddleware来定义一个中间件

    • 主要有两个参数：to、from，分别是要前往的页面的路由信息和，当前页面的路由信息
    • 返回值
      • navigateTo()：用于设置要前往的页面
      • abortNavigation(err?: string | Error)：用于停止当前导航，可以传递一个错误信息

  • 例

    typescript
    export default defineNuxtRouteMiddleware((to, from) => {
  console.log('要去那个页面:' + to.path);
  console.log('来自那个页面:' + from.path);

  if (to.path === '/about') {
    return abortNavigation();
  } else {
    return navigateTo('/');
  }
});
