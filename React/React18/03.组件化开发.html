组件化开发 ​

01. 基础知识 ​

（1）组件的分类 ​

  • 根据定义方式

    • 函数组件（Functional Component）
    • 类组件（Class Component）

  • 根据状态

    • 无状态组件（Stateless Component）
    • 有状态组件（Stateful Component）

  • 根据职责

    • 展示型组件（Presentational Component）
    • 容器型组件（Container Component）

（2）组件的定义 ​

  • 数据逻辑和 UI 展示分离

    • 函数组件、无状态组件、展示型组件主要关注 UI 的展示
    • 类组件、有状态组件、容器型组件主要关注数据逻辑

  • 组件命名

    • 以大写字符开头

（3）组件的渲染 ​

  • 组件的渲染主要是指render()函数的返回值

  • render()函数执行时，会检测this.props和this.statae，并返回以下类型之一：

    • React元素：返回单个元素
    • 数组或Fragments：返回多个元素
    • Portals：可以渲染子节点到不同的 DOM 子树中
    • 字符串或数值类型：被渲染为文本节点
    • 布尔类型或null：什么都不渲染

02. 类组件和函数式组件 ​

（1）类组件 ​

  • 类组件需要继承自：React.Component或者React.PureComponent

    • React.Component：普通组件
    • React.PureComponent：SCU优化后的组件

  • 代码示例

    jsx
    // 默认导出
// import React from 'react'
// 按需引入
import { PureComponent } from 'React';

class App extends PureComponent {
  // 可选
  constructor() {
    super(); // 必须
    this.state = {}; // 数据
  }
  // 必选
  render() {
    return <div>类组件</div>;
  }
}

export default App;

03. 生命周期 ​

（1）基本介绍 ​

这里的生命周期主要是类组件的生命周期

  • constructor：在组件实例创建时立即调用
    • 可以在这里初始化数据this.state
    • 可以在这里给事件绑定实例（bind）
    • 如果不写也会默认执行
  • componentDidMount：在组件挂载到 DOM 后立即调用
    • 可以在这里对 DOM 进行操作
    • 可以在这里发送网络请求
    • 可以在这里添加一些全局监听事件
  • componentDidUpdate：在数据更新后立即调用，首次渲染时不会执行
    • 当组件更新后，可以在此处对 DOM 进行操作
    • 可以对更新前后的数据进行比较
    • 可以在这里进行网络请求
  • ComponentWillUnmount：在组件卸载或销毁之前直接调用
    • 可以执行一些清理操作
  • shouldComponentUpdate：自行定义是否更新组件，重新渲染
    • 用于性能优化的，某些场景下不需要重新渲染

（2）执行过程 ​

  • Mounting 阶段
    • constructor -> getDeriveStateFromProps -> render -> componentDidMount
  • Updating 阶段
    • getDeriveStateFromProps -> shouldComponentUpdate -> render -> getSnapshotBeforeUpdate -> componentDidUpdate
  • Unmounting 阶段
    • componentDidUnmount

（3）SCU ​

  • 在React中，只要数据发生变化，所有组件都会重新渲染，即调用render函数

    • 但实际上应该是组件依赖的props | state发生变化时，才需要去调用render函数
    • 这个时候，可以使用shoulComponentUpdate生命周期函数，来判断是否需要调用当前组件的render方法
    • 这种性能优化的方法就是SCU

  • SCU可以接受两个参数，并且必须要有返回值

    • 两个参数：
      • 参数一：最新的props属性
      • 参数二：最新的state属性
    • 返回值：
      • 返回值是一个boolen类型，true为调用render，false为不调用render

    如果不写这个生命周期，则默认返回为true，即所有组件都会重新渲染

  • 主要作用：

    • 即，通过判断前后props | state是否相同，来返回true | false
    jsx
    shouldComponentUpdate(nextProps, nextState) {
  if(this.props.message !== nextProps.message) {
    return true
  }else {
    return false
  }
}

04. 组件通信 ​

注：子组件是无法直接修改父组件传递过来的数据的

（1）父传子 ​

  • 父组件通过标签属性的形式将数据传递给子组件

  • 子组件通过props参数获取父组件传递的数据

    JSX
    // 父组件
class Father extends PureComponent {
  render() {
    return (
      <div>
        <Child message="message"/>
      </div>
    )
  }
}

// 子组件 -- 类组件
class Child extends PureComponent {
  // 父组件传递的参数 将会以对象形式 作为 constructor 的参数
  constructor(props) {
    // 可以将其传递给 父类，会自动挂载到 this.props 上
    super(props)

    // 或者，可以自己手动将其挂载到 子类
    // this.props = props
  }
  // 如果 constructor 中不需要额外处理其它数据，则这里也可以不写，会自动执行上面的代码
}

// 子组件 -- 函数式组件
export default function Child(props) {
  // 父组件传递的参数将会作为函数的参数传递进来
  return <h2>{props.message}</h2>
}

（3）子传父 ​

  • 父组件给子组件传递一个回调函数

  • 子组件调用父组件传递的回调函数，并将需要传递的参数通过这个回调函数传回去

    jsx
    class Father extends PureComponent {
  // 由子组件调用父组件的这个方法，会接收到子组件传递过来的参数
  getChildProps(props) {
    console.log('child props:', props);
  }
  render() {
    return (
      <div>
        {/* 将方法也作为参数传递给 子组件 */}
        <Child
          message="message"
          getChildProps={() => {
            this.getChildProps();
          }}
        />
      </div>
    );
  }
}

class Child extends PureComponent {
  handlePostParams() {
    // 在子组件内调用父组件传递的方法，可以将参数传递给父组件
    this.props.getChildProps('message');
  }
}
Child.propTYpes = {
  getChildProps: propTypes.func,
};

05. React中的插槽 ​

  • React中是没有插槽的，但是可以通过其它方法来实现

（1）以children属性传递 ​

  • 直接写在子组件标签之间的内容，会通过props属性的children属性传递到子组件

    • 如果传入一个子元素，则这个children是一个对象
    • 如果传入多个子元素，则children是一个数组
    jsx
    // children 插槽
class Father extends PureComponent {
  render() {
    return (
      <Child>
        <div>这里的内容会作为子组件的 props.children 属性</div>
      </Child>
    );
  }
}

class Child extends PureComponent {
  rernder() {
    // 父组件中 写在子组件标签之间的内容 会被放入子组件的 children 属性上
    const { children } = this.props;

    return (
      <div>
        {/* 这个 children 属性，可以是一个数组，也可以直接是一个对象, 由父组件内传入的内容决定 */}
        {/* 可以将 children 属性内部的东西取出来，作为插槽内容 */}
        {/* 单个，直接使用 */}
        {children}
        {/* 多个，循环取出 */}
        {children.map((item) => (
          <span>{item}</span>
        ))}
      </div>
    );
  }
}

（2）以props直接传递 ​

  • 直接将React元素通过props属性传入

    • 在子组件通过对应属性取出然后使用
    • 可以将React元素以函数的形式传入，实现类似作用域插槽的效果
    jsx
    // props 插槽
class Father extends PureComponent {
  const leftSlot = <span>Hello</span>;
const rightSlot = <span>World</span>;

render() {
  return (
    <Child leftSlot={leftSlot} rightSlot={ (item) => <span>{item}</span>} />
  )
}
}

class Child extends PureComponent {
  rernder() {
    // 由父组件将 组件传递给子组件，然后由子组件使用
    const { leftSlot, rightSolot } = this.props

    return (
      <div>
        { /* 普通插槽 实现方式 */ }
        <div>{leftSlot}</div>
        { /* 作用域插槽 实现方式 */ }
        <div>{ rightSolot('由这里决定内容是啥') }</div>
      </div>
    )
  }
}
