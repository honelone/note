React 中的 CSS ​

01. 简要概述 ​

  • 首先，CSS 的设计并不是为组件化而生的，组件化的 CSS 需要符合以下条件
    • 局部 CSS
    • 动态 CSS
    • 支持所有 CSS 特性
    • 符合 CSS 风格
  • React中的 CSS，并没有统一样式风格，所以解决方案也很多

以下为React中的几种解决方案

02. 内联样式 ​

  • 在style属性里，以小驼峰命名属性的Javascript对象

    • 可以动态获取当前state中的状态，来动态设置样式

  • 缺点：

    • 写法上需要使用小驼峰标识
    • 一些样式没有提示
    • 代码混乱、臃肿
    • 某些样式无法编写，如伪类、伪元素

03. CSS 单文件 ​

  • 单独编写一个 CSS 文件，然后在需要使用的文件中引入

  • 缺点：

    • 这种 CSS 属于全局的 CSS，样式之间会相互影响

上面两种方案适用于简单的小项目

下面两种方案为主要解决方案

04. CSS Module ​

  • 使用webpack进行配置，React脚手架已经内置了CSS Modules的配置

  • 使用方式：

    • 使用方式同 CSS 单文件
    • 但需要将样式文件名后跟上.module关键字
    • 如index.css -> index.module.css，App.less -> App.module.less
    • 这样该样式文件只会在引入的文件中生效，不会影响全局的样式

  • 优点：

    • 解决了 CSS 单文件的作用域问题

  • 缺点：

    • 引用的类名，不能使用连接符，在Javascript中是不识别的
    • 所有的className都必须使用{style.className}的形式来编写
    • 不方便动态修改某些样式，一些样式依然需要使用内联样式

05. CSS in JS ​

  • CSS-in-JS，是指一种模式

    • 将 CSS 写入到Javascript中的方式，可以方便的使用 JS 的状态
    • 其 CSS 由Javascript生成而不是在外部文件中定义

  • 此功能由第三方库提供：

    • 比较流行的库：styled-components、emotion、glamorous
    • 拥有类似于 CSS 预处理器一样的样式嵌套、函数定义、逻辑复用、动态修改状态等功能

（1）标签模板字符串 ​

  • 正常情况下，函数的调用是通过函数名()的方式进行调用的，但还有另一种调用方式，即模板字符串的调用方式

    js
    function foo(...args) {
  console.log(args);
}

const name = 'React';
const nmum = 18;

// 1. 正常调用
foo('Hello, World'); // log: ['Hello, World']

// 2.模板字符串调用

// -- 这里内部没有插入变量，所以只有一个元素
foo`Hello, World`; // log: [['Hello, World']]

// -- 这里会根据 插入的变量 拆分模板字符串 ,注意：最后一个变量后面会有空字符串
foo`Hello, ${name}`;
// log: [['Hello, ', ''], 'React']

foo`Hello, ${name}, Hello, ${num}`;
// log: [['Hello, ', ', Hello, ', ''], 'React', 18]

// 调用之后，模板字符串会被拆分
// 第一个元素是：数组。是由模板字符串拆分的 字符串组合, 最后一个变量后面会被拆分为 空字符串
// 后面的元素是：模板字符串的变量内容顺序显示

（2）styled-components ​

  • 主要是通过ES6的标签模板字符串的语法实现

    • vscode-styled-components插件可以提供更好的语法高亮提示

  • 创建

    jsx
    // style.js
import styled from 'styled-components';

// 表示创建一个 div 组件， 并通过标签模板字符串的方式调用
export const AppWrapper1 = styled.div``;

// 1. 在模板字符串中，可以像 css 一样正常写样式
// -- 表示为 AppWrapper 包裹的子元素写样式
// -- 内部支持样式嵌套
export const AppWrapper2 = styled.div`
  background-color: #fff;
  .class1 {
    color: red;
    &:hover {
      color: orange;
    }
  }
`;

// 2. 支持外部传入数据，通过 props 接收
// -- 或者从一个 全局的样式变量文件 引入其它变量，可以直接使用
export const AppWrapper3 = styled.div`
  .class2 {
    color: ${(props) => props.color};
    font-size: ${(props) => props.size}px;
  }
`;

// 3. 如果没有传入参数，可以通过 .attrs() 设置默认值, 需要传入一个函数，传入对象有问题
export const AppWrapper4 = styled.div.attrs((props) => {
  console.log(props);
  return {
    color: props.color || 'red',
    size: 30,
  };
})`
  .class2 {
    color: ${(props) => props.color};
    font-size: ${(props) => props.size}px;
  }
`;

// 4. 支持样式继承，可以从另一个 样式组件 继承其样式
export const AppWrapper5 = styled(AppWrapper4)`
  .class2 {
    color: ${(props) => props.color};
    font-size: ${(props) => props.size}px;
  }
`;

08. 过渡动画（简单了解，用时再补） ​

  • 需要引入第三方库：react-transition-group

    • 可以实现组件的入场和离场动画

  • 这个库主要包含四个组件

    • Transition：过渡动画
    • CSSTransition：过渡动画
    • SwitchTransition：用于两个组件切换显示、隐藏
    • TransitionGroup：包裹多个动画

（1）CSSTransition ​

  • 三种状态

    • 开始状态：-appear、-enter、exit
    • 执行动画：-appear-active、-enter-active、-exit-active
    • 执行结束：-appear-done、-enter-done、exit-done

  • 属性

    • in：
      • 为true，则触发进入状态。-enter、-enter-active
      • 为false，则触发退出状态。-exit、-exit-active
    • classNames
      • 添加在上面的三种状态之前的名称，用于自定义各种状态的动画
    • timeout：过渡时间
    • appear：是否在初次进入添加动画，需要in属性配合
    • unmountOnExit：退出后卸载组件

  • 钩子函数

    • onEnter：进入动画之前被触发
    • onEntering：进入动画时被触发
    • onEntered：进入动画结束后被触发

  • 代码示例

    jsx
    <CSSTransition in={true} classNames="self" unmountOnExit={true} timeout={2000}>
  ...
</CSSTransition>

（2）SwitchTransition ​

  • 属性

    • mode
      • in-out：表示新组建先进入，旧组件再移除
      • out-in：表示旧组件先移除，新组件再进入

  • SwitchTransition组件内部

    • 不能直接包裹需要切换的组件
    • 需要使用CSSTransition组件或者Transition组件包裹
    • 且这两个组件不能使用in属性来判断元素的状态，而是使用key属性，key需要唯一

  • 代码示例

    jsx
    <SwitchTransition mode="in-out">
    <CSSTransition key={}>
    	...
    </CSSTransition>
</SwitchTransition>

（3）TransitionGroup ​

  • 当需要使用一组动画的时候就需要将CSSTransition组件都放入Transition组件中

    • 同样的，CSSTransition不能使用in属性来判断状态，而是需要使用key

  • 属性

    • component：表示当前TransitionGroup被渲染为哪个元素

  • 代码示例

    jsx
    <TransitionGroup component="ul">
  <CSSTransition key="child1">
    <Child1 />
  </CSSTransition>
  <CSSTransition key="child2">
    <Child2 />
  </CSSTransition>
</TransitionGroup>
