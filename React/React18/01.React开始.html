React 开始 ​

01. 重要性 ​

  • React——用于构建用户界面的Javascript库

  • 前端流行的三大框架

    • Angular：门槛高，国内市场低
    • Vue：国内占有率最高
    • React：占有率高，必学框架

  • Vue设计之初，很多借鉴Angular和React

    • Vue Composition Api学习了React Hooks

  • Flutter也借鉴了React

    • Flutter中的Widget - Element - RenderObject
    • 对应React中的JSX - 虚拟DOM - 真实DOM

  • 可以说React是前端的先驱者，引领前端的潮流

02. 特点 ​

  • 声明式编程：只需要维护自己的状态，当状态改变时，React可以根据最新的状态去渲染我们的 UI 界面

  • 组件化开发：页面的每个部分都是一个组件

  • 多平台适配：React - Web页面、ReactNative - 移动端跨平台、ReactVR - 虚拟现实

    额外的，要想学好SSR，Node.js也得学好，因为SSR是基于Node.js的

  • React是MVC单向数据驱动框架

    • M：model数据层
    • V：view视图层
    • C：controller控制层

03. 开发依赖 ​

  • 开发React必须依赖三个库

    • react：核心代码
    • react-dom：react渲染在不同平台所需要的核心代码
    • babel：将JSX转换成React代码的工具

    每个库都只会做自己的事

  • 这是因为React核心代码中是包含了react-web和react-native的代码

    • react-dom会针对web和native所完成的事情不同，
    • 对于web端，会将JSX渲染成真实 DOM
    • 对于native端，会将JSX渲染成原生的控件

  • Babel

    • 默认情况下，开发React可以不使用babel
    • 但需要自己使用React.createElement来编写源代码
      • 这种方法编写代码非常繁琐，可读性差
    • 所以我们需要使用Babel来将我们编写的JSX代码转换成React.createElement代码

  • 使用方式

    • CDN引入
    • 下载本地引入
    • npm引入

  • 事件

    • 在类中直接定义一个函数，并且将这个函数绑定到元素的onClick事件上
    • 注意，这个函数的this默认指向的是undefined
      • 这是因为正常 DOM 操作中，函数中的this其实是节点对象
      • 但由于React并不是直接渲染成真实的DOM，它其实是React的Element对象
      • 函数执行时，并没有绑定this，所以默认情况下就是一个undefined
      • 所以在这个函数中，如果要使用当前对象，就需要手动去改变this的指向
    js
    // 假设我们这里有个类
const app = new App();
app.func(); // 默认的这个方法里面的 this 是指向这个实例化的对象 app
// 但是当我们赋值给其它变量时
const foo = app.func;
foo(); // 这里的 foo 由 windows 调用，所以内部的 this 指向的是 window,如果是严格模式 this == undefined

class MyComponent extends React.Component {
  // 自定义方法
  myFunc() {
    this.setState({
      message: 'Hello, React!',
    });
    console.log(this);
  }
  render() {
    // 那么这里的情况和上面类似 实际上就等于  const onClick = this.myFunc()
    // -- 所以这里的点击事件会丢失 this 的指向
    return <div onClick={this.myFunc}></div>;

    // -- 所以这里要手动的将其 this 指定一下，将其 this 绑定为当前对象
    return <div onClick={this.myFunc.bind(this)}></div>;
  }
  constructor() {
    // 另一种是提前在 constructor 中绑定好, 这样在 render 中可以直接使用把
    this.myFunc = this.myFunc.bind(this);
  }
}

// cracro : create react app config

// 安装： npm i -D @craco/craro@alpha

// 配置： craro.config.js
const path = require('path')
const resolve = pathname => path.resolve(__dirname, pathname)

// 安装： npm i carco-less
const CracoLessPlugin = require('carco-less')

module.exports = {
  plugins: [
    plugin: CracoLessPlugin
  ],
  webpack: {
    alias: {
      '@': resolve('src'),
      'components': resolve('src/components')
    }
  }
}

// 启用： carco start

const Home = memo(() => {
  // ...

  useEffect(() => {
    console.log('加载');
    setData('Hello, World!');

    return () => {
      console.log('卸载');
    };
  }, [data]);

  return (
    <div>
      <h2>Home</h2>
      <div>{data}</div>
    </div>
  );
});

export default Home;
