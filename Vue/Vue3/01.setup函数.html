setup()函数 ​

01. setup() ​

（1）基本概念 ​

  • setup()函数是一个新的组件选项，同时也是Composition API的入口函数

  • 任何Option API都可以写在setup()函数内部，但都需要按需引入

  • 需要注意的是：

    • 所有在setup()函数内部定义的数据都需要通过return返回，才能在<template>中使用
    • setup()函数在beforeCreate之前执行，此时组件对象还没有创建，所以无法访问this
    js
    import { defineComponent } from 'vue';

export default defineComponent({
  setup() {
    const msg = 'This is a message';

    // return 中返回的数据 才能在 template 中使用
    return {
      msg,
    };
  },
});

02. setup()的两个参数 ​

setup()函数可以接收两个参数

  • props：用于接收外部props选项中的所有属性
  • context：用来表示当前上下文对象

（1）props参数 ​

  • props：作为setup()函数的第一个参数传入，用于在setup()函数中接收外部props属性中的参数

    • 也就是只有在外部的props属性中接收了的参数，才能在setup()函数中的props参数中使用
    js
    import { defineComponent } from 'vue';

export default defineComponent({
  props: {
    msg: {
      type: String,
      default: 'This is a message',
    },
  },
  setup(props) {
    console.log(props.msg);
  },
});

（2）props参数的类型 ​

  • props参数的类型需要在外部的props选项中进行定义，基本定义方式同Vue2中一致

  • 基本使用方式：

    • 直接使用类型定义
      • 类型名称首字母大写
      • 多个类型使用管道符|分隔
    • 使用对象形式
      • type：用于定义类型
      • default：用于设置默认值
        • 对于复杂数据类型，需要以函数形式返回
      • required：用于设置是否为必传属性
    js
    import { defineComponent } from 'vue';

export default defineComponent({
  props: {
    num: Number,
    arg: Number | String,
    msg: {
      type: String,
      default: 'This is s message',
      required: true,
    },
    arr: {
      type: Array,
      default: () => {
        return [];
      },
    },
  },
});

（3）context参数 ​

  • context：作为setup()函数的第二个参数传入，其内部暴露了 4 个属性（函数）

    • attrs：用于接收，所有从父组件传进来的，但没有在props属性中接收的属性，作用同 $attrs

    • emit：用于向父组件派发事件

    • slots：用于接收所有传入的插槽内容的对象

    • expose：用于向外部暴露当前组件的expose出去的属性

      这四个属性可解构使用

    js
    import { defineComponent } from 'vue';

export default defineComponent({
    slots: {
        header: { foo: string; bar: number }
    },
    emits: {
        updateMsg: (val: string) => void
    },
    setup(props, { attrs, slots, emit, expose }) {

        // 透传 Attributes（非响应式的对象，等价于 $attrs）
        console.log(attrs)

        // 插槽（非响应式的对象，等价于 $slots）
        console.log(slots)

        const msg = 'This is a message'

        // 触发事件（函数，等价于 $emit）
        emit('updateMsg', msg)

        // 暴露公共属性（函数）
        expose({
            msg: msg
        })
    }
})

（4）$attrs ​

  • Vue3中仍然是保留着$attrs属性的，但与Vue2相比：

    • $atttrs现在包含了传递给组件的所有属性，包括class和style，（Vue2 中是不包含的）

    • 一般来讲，这些属性会自动继承到根元素上

      • 除非指定了inheritAttrs: false，

    • 但如果有多个根元素，就需要主动去指定了，比如

      vue
      <template>
  <div :class="$attrs.class">1</div>
  <div :style="$attrs.style">2</div>
</template>
      1
      2
      3
      4

  • 另外，$listeners属性已废除，并移入了$attrs中，所以可以直接通过$attrs来调用父组件上v-on事件监听器

    • 需要注意的是，调用的时候需要加上on前缀，同onClick
    js
    import { defineComponent } from 'vue';

export default defineComponent({
  setup(props, { attrs }) {
    attrs.onUpdate(); // <Child @update />
  },
});

  • 注：

    • $attrs对象包含了除组件所声明的 props 和 emits 之外的所有其他attribute，例如 class，style，v-on 监听器等等
    • 透传的attributes在 JavaScript 中保留原始大小写
    • v-on 事件监听器会被暴露为一个函数，如@click暴露为$attrs.onClick
    • 可以使用v-bind="$attrs"将父组件的所有透传的属性传递给子组件
      • 没有参数的 v-bind会将一个对象的所有属性都作为 attribute 应用到目标元素上

03. <script setup>语法糖 ​

（1）基本概念 ​

  • 任何在 <script setup> 声明的顶层的绑定都能在模板中直接使用
    • 包括变量，函数声明，以及 import 引入的内容
    • 不需要再额外return返回
    • 引入的组件也可以直接使用
    • 组件名即为文件名

（2）defineProps ​

  • 用于声明 props，同setup函数中的props参数，不需要额外引入

  • 普通形式

    js
    // <script setup>

// 定义 props ，类型定义同外部 props 选项
const props = defineProps({
  msg: {
    type: String, // 类型首字母大写
    required: true,
    default: 'This is a message',
  },
  arr: {
    type: array,
    default: () => [],
  },
});

// 使用
console.log(props.msg);

（3）defineEmits ​

  • 用于声明向父组件派发的事件，同setup函数中的context.emit，不需要额外引入

    typescript
    // 定义 emits, 用于定义需要向父组件派发的事件
const emits = defineEmits(['change', 'update']);

// 向父组件派发事件
emits('change');

// 携带参数
emits('update', 'bar');

// 基于类型的 emits
const emits = defineEmits<{
  (e: 'change'): void;
  (e: 'update', value: string): void;
}>();

// 3.3+：另一种更简洁的语法
const emit = defineEmits<{
  change: [id: number]; // 具名元组语法
  update: [value: string];
}>();

（5）useAttrs ​

（7）配合<script>标签 ​

  • <script setup>可以同时搭配<script>标签使用

  • 如果设置了lang，则两个<script>标签的lang需要保持一致

  • 在这个普通的<script>标签内可以配置一些<script setup>无法配置的选项

    typescript
    import { defineComponent } from vue;
export default defineComponent({
    name: 'MyComponent', // 配置组件名
    inheritAttrs: false, // 配置是否禁用属性继承，默认值为 true
})

（10）defineModel(3.4) ​

  get() {
    return props.name;
  },
  set(newVal) {
    emits('update:name', newVal);
  },
});

// 现在
const currentName = defineModel<string>('name', { required: true });

04. setup中的钩子函数 ​

（1）生命周期函数 ​

  • 在setup()函数中使用生命周期钩子函数，需要额外引入相应的 API

    js
    // <script setup>

import {
  onBeforeMount,
  onMounted,
  onBeforeUpdate,
  onUpdated,
  onBeforeUnmount,
  onUnmounted,
  onErrorCaptured,
} from 'vue';

// 挂载前
onBeforeMount(() => {
  console.log('beformounted!');
});
// 挂载后
onMounted(() => {
  console.log('mounted!');
});

// 更新前
onBeforeUpdate(() => {
  console.log('beforupdated!');
});
// 更新后
onUpdated(() => {
  console.log('updated!');
});

// 销毁前
onBeforeUnmount(() => {
  console.log('beforunmounted!');
});
// 销毁后
onUnmounted(() => {
  console.log('unmounted!');
});

// 捕捉到错误
onErrorCaptured(() => {
  console.log('errorCaptured!');
});

// 新增的onRenderTracked、onRenderTriggered
