01. 布局系统layouts ​

（1）基础布局 ​

  • 在 layouts 下创建的布局都可以应用到页面上

  • 例如，创建 layouts/custom.vue 布局组件

vue
<!-- layouts/custom.vue -->
<template>
  <!-- 根元素是必需的 -->
  <div>
    这里是使用当前布局会共享的内容
    <slot />
    <!-- 页面出口 -->
  </div>
</template>

  • 在页面中使用，在 /home 页面中使用，有两种方式
    • 使用layout属性，需要在组件选项中配置
    • 使用<NuxtLayout>组件，该组件全局可用
vue
<!-- pages/home.vue -->
<!-- 方式一 -->
<script>
  export default {
    layout: 'custom', // 属性值为 布局组件名字
  };
</script>

<!-- 方式二 -->
<template>
  <!-- name 为布局组件的名字 -->
  <NuxtLayout name="custom">home</NuxtLayout>
</template>

02. 路由系统pages ​

说一点，虽然Vue3支持多个根元素，但仍然建议在Nuxt3中使用一个根元素，即同Vue2使用方法一致

（1）NuxtPage ​

  • 在 app.vue 中加入 NuxtPage

    • 这个组件会自动收集pages下面的页面并生成路由
    vue
    <template>
  <NuxtPage />
</template>
    1
    2
    3

  • 例如，/ 页面只需创建 pages/index.vue

    vue
    <!-- pages/index.vue -->
<template>default</template>
    1
    2

  • 又例如，/home 页面只需创建 pages/home.vue

    vue
    <!-- pages/home.vue -->
<template>home</template>
    1
    2

（2）嵌套路由 ​

  • 如果需要子路由，可以采用目录嵌套的方式

  • 例如，/user/details 页面，只需创建 pages/user/details.vue

    vue
    <!-- pages/user/details.vue -->
<template>user/details</template>
    1
    2

  • 当然你还可以创建 pages/user.vue 来做成嵌套路由

    • 需要配合<NuxtChild>组件，同<RouterView>
    html
    <!-- pages/user.vue -->
<template>
  这里是所有子组件共享的内容
  <NuxtChild />
  <!-- 子路由出口 -->
</template>

（3）动态路由 ​

  • 如果需要动态路由，可以使用 [param] 的方式

  • 例如，/user/:id 页面只需创建 /pages/user/[id].vue

    • 同时可以通过 $route.params.id 获取到具体的参数
    vue
    <!-- pages/user/[id].vue -->
<template>user / {{ $route.params.id }}</template>
    1
    2

  • 当然父路由也可以是动态的

    • 例如，/user-:group/:id 页面只需创建 /pages/user-[group]/[id].vue
    • 同时可以通过 $route.params.group 获取到具体的参数
    vue
    <!-- /pages/user-[group]/[id].vue -->
<!-- 路由 /user-:group/:id 将渲染 -->
<template>user-{{ $route.params.group }} / {{ $route.params.id }}</template>

    2

（4）路由导航 ​

  • 可以使用NuxtLink 用进行页面之间的导航，同<RouterLink>

    • 对于内部链接（站点内的页面），应该使用<NuxtLink>进行导航
    • 对于外部链接（站点外的页面），应该使用<a>标签进行导航，虽然也可以使用<NuxtLink>
    vue
    <!-- app.vue -->
<template>
  <!-- 点击将跳转到 /about 页面 -->
  <NuxtLink to="/about">内部链接</NuxtLink>
  <!-- 点击将跳转到 外链 -->
  <a href="https://other.com">外部链接</a>
</template>

（5）路由元信息 ​

  • 可以通过 definePageMeta 来设置当前路由的元信息

    vue
    <script setup lang="ts">
  // 定义元信息
  definePageMeta({
    foo: 'bar',
  });

  // 获取当前路由
  const route = useRoute();

  // 获取元信息
  const { meta } = route;
  console.log(
    toRaw(meta), // 输出 { foo: 'bar' }
  );
</script>

03. 组件复用components ​

（1）自动引入 ​

  • 在 components 目录下的组件将被自动引入

    • 例如，components 下有个 bar 组件
    vue
    <!-- components/bar.vue -->
<template>Bar</template>

<!-- app.vue -->
<template>
  <Bar />
  <!-- bar 组件将被按需自动引入并注册 -->
</template>

（2）嵌套引入 ​

  • 如果组件嵌套在目录内，组件的名称将基于自己的路径和文件名，以驼峰形式引入，如果有重复的地方，则会删除重复的段

    • 例如，components/user 下有个 avatar 组件，components/base/foo/foo/下有个button组件
    html
    <!-- components/user/avatar.vue -->
<template>user avatar</template>

<!-- components/base/foo/foo/button.vue -->
<template>base foo button</template>

<!-- app.vue -->
<template>
  <UserAvatar />
  <!-- avatar 组件将以 驼峰形式 被按需自动引入并注册 -->
  <BaseFooButton />
  <!-- button 组件将会去掉重复端，然后以 驼峰形式 注册引入 -->
</template>

（3）动态引入 ​

  • 要动态导入一个组件 ，或者说是懒加载一个组件，只需要在原组件名前加上 Lazy 前缀

    html
    <!-- app.vue -->
<template>
  <LazyUserAvatar />
  <!-- avatar 组件将会 动态加载 -->
</template>
