02. <Teleport>传送门 ​

  • teleport可以将内部的组件挂载到另一个位置

  • 包含两个属性

    • to：表示要挂载到的目标元素，使用CSS选择器
    • disabled：表示是否禁用teleport的功能，如果禁用，则其插槽内容不会移动到任何位置
    html
    <template>
    <!-- 将被传送挂载到 #app 节点下 -->
    <teleport to="#app">
        <div>AAA</div>
    </teleport>

    <!-- 将被传送挂载到 .class 节点下 -->
    <teleport to=".class">
        <div>BBB</div>
    </teleport>
</template>

03. <Suspense>未完待续 ​

  • <suspense>组件有两个插槽

    • #default：在内部组件解析完毕后显示
    • #fallback：在内部组件解析完毕前显示
    • 且，异步组件不需要作为直接子节点，只需要包裹在里面即可
html
<template>
    <suspense>
        <template #default>
            <!-- 默认插槽内容 -->
        </template>
        <template #fallback>
            <div>
                Loading...
            </div>
        </template>
    </suspense>
</template>

04. <Transition>动画组件 ​

  • 用于控制内部组件的进入和离开动画

    • 这个进入和离开可以由这些动作触发：v-if、v-show、<component :is="current" />触
    • 内部必须是单组件，或者说只能有一个根元素

  • 动画样式

    • v-enter-from：
      • 进入动画的起始状态
      • 在元素插入之前添加，在元素插入完成后的下一帧移除
    • v-enter-active：
      • 进入动画的生效状态，应用于整个进入动画阶段
      • 在元素被插入之前添加，在过渡或动画完成之后移除
      • 这个 class 可以被用来定义进入动画的持续时间、延迟与速度曲线类型
    • v-enter-to：
      • 进入动画的结束状态
      • 在元素插入完成后的下一帧被添加 (也就是 v-enter-from 被移除的同时)，在过渡或动画完成之后移除
    • v-leave-from：
      • 离开动画的起始状态
      • 在离开过渡效果被触发时立即添加，在一帧后被移除
    • v-leave-active：
      • 离开动画的生效状态，应用于整个离开动画阶段
      • 在离开过渡效果被触发时立即添加，在过渡或动画完成之后移除
      • 这个 class 可以被用来定义离开动画的持续时间、延迟与速度曲线类型
    • v-leave-to：
      • 离开动画的结束状态
      • 在一个离开动画被触发后的下一帧被添加 (也就是 v-leave-from 被移除的同时)，在过渡或动画完成之后移除

  • 过渡效果命名

    • <Transition>：未命名，则其过渡效果为：v开头的类
    • <Transition name="fade">：命名为fade，则其过渡效果为：fade开头的类。即将上述类名的v替换为fade

  • 自定义过渡class

    • 通过响应属性可以覆盖默认的过渡效果类
    vue
    <!-- 假设你已经在页面中引入了 Animate.css -->
<Transition
  name="custom-classes"
  enter-active-class="animate__animated animate__tada"
  leave-active-class="animate__animated animate__bounceOutRight"
>
  <p v-if="show">hello</p>
</Transition>

10. <style>拓展 ​

（1）三个选择器 ​

  • 深度选择器
    • 使用:deep()伪类，可以影响到子组件内部的样式
    • 通常也使用这个方法来改变UI框架组件的样式，如果无法改变，那么大概率是样式层级的问题
  • 全局选择器
    • 使用:global()伪类，可以让当前样式作用到全局
    • 慎用，可能会造成其它页面的样式错乱
  • 插槽选择器
    • 使用:slotted()伪类，可以确切地将插槽内容作为选择器的目标
    • （暂时还未用到，用到了再来补充）

（2）scoped属性 ​

  • 使用scoped属性可以让样式只应用在当前文件
    • 但它还是会影响子组件的根节点的样式
  • 可以混合使用作用域样式<sctyle scoped>和非作用域样式<style>
    • 即，可以在同一个组件中，同时包含上面两种形式的样式

（3）module属性 ​

  • 在<style>标签上可以增加module属性，即<style module>

    • <style module>会被编译为CSS Module，并且将生成的CSS类作为$style对象的键暴露给组件
    • 可以直接在<template>中使用$style

  • 另外，还可以主动设置 module 属性来自定义注入名称，即<style module="myModule">

    • 这样编译后的CSS类会作为myModule对象的键暴露出去

  • 如果要在<style>标签内使用，则需要额外引入useCssModule

    html
    <!-- 默认暴露的键为 $style -->
<style module>
    .red {
        color: red;
    }
</style>

<!-- 暴露的键为设置的名称 myModule -->
<style module="myModule">
    .blue {
        color: blue;
    }
</style>

<script setup>
    // script 内部使用需要引入 useCssModule 
    import { useCssModule } from 'vue'

    // -- 不传参，表示获取默认的 CSS Module
    const defaultStyle = useCssModule()
    console.log(style.className)

    // -- 传参，表示获取指定的 CSS Module
    const myModuleStyle = useCssModule('myModule')
    console.log(myModuleStyle.className)

</script>

<template>
    <p :class="$style.red">
        This color is red
    </p>
    <p :class="$myModule.blue">
        This color is blue
    </p>
</template>

（4）状态驱动的动态CSS ​

  • 在 <style> 标签里，现在可以通过 v-bind() 将 CSS 的值关联到动态的组件状态上

    vue
    <template>
	<p>hello</p>
</template>

<script setup>
    const theme = {
        color: 'red'
    }
</script>

<style scoped>
    p {
        color: v-bind('theme.color');
    }
</style>
