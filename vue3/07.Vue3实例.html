01. 创建实例 ​

  • 通过createApp函数来创建一个应用实例

    typescript
    // 需要单独引入函数
import { createApp } from 'vue'

// 引入根组件
import App from './App.vue'

// 创建实例
const app = createApp(App)

02. 实例的挂载 ​

  • 通过mount函数将当前应用程序实例挂载在一个容器上

    • 这个容器可以是一个DOM元素（Element）

  • 或者一个CSS选择器（string）

    typescript
    app.mount('#app')
    1

  • 通过unmount函数将卸载一个已经挂载好的应用实例

    • 同时，还会触发所有应用组件上的所有组件的onUnmounted生命周期
    typescript
    1

app.unmount()

04. 应用配置 ​

在应用实例app上，会暴露一个config对象，这个对象上允许我们进行一些应用级别的公共配置

（1）错误处理 ​

  • 用于捕获所有由子组件抛出的而未被捕获处理的错误

    typescript
    app.config.errorHandler = (err) => {
    // 处理错误的具体方法
}
    1
    2
    3

（2）警告处理 ​

  • 用于开发环境的Vue运行时的警告处理，在正式环境不生效

    typescript
    app.config.warnHandler = (msg) => {
    // 警告处理
}
    1
    2
    3

（3）全局属性 ​

  • 用于在app应用实例上挂载全局的属性、方法

    typescript
    // Vue2 -- 挂载到 Vue 的原型上
Vue.prototype.$http = () => {...}

// Vue3 -- 挂载到 app 实例上
app.config.globalProperties.myProperty = 'myProperty'

app.config.globalProperties.myMethod = () => {}

app.config.globalProperties.$message = message

（4）getCurrentInstance ​

  • getCurrentInstance：用于获取当前的vue实例，这个实例可以获取到所有挂载在app上的属性和方法

    js
    import { getCurrentInstance } from 'vue'

// 使用注册的全局方法 -- 另一种方法是 provide/inject
const { proxy }: any = getCurrentInstance()

// console.log(proxy.$message)
proxy.$message.info('This is a normal message')
