01. 定义路由 ​

  • 需要引入createRouter()函数

  • 路由模式变化

    • 参数：mode => history
    • 模式：
      • hash模式：createWebHashHistory()
      • history模式：createWebHistory()
      • abstract模式：createMemoryHistory()

  • base配置

    • base配置现作为路由模式的第一个参数进行传递

  • 自定义内容变化

    • 只能定义在meta属性中
    js
    // 引入路由方法
import { createRouter, createWebHashHistory, createWebHistory } from 'vue-router'

// 定义路由，每个路由都需要映射到一个组件
const routes = [
    {
        path: '/',
        component: () => import('@/components/Home.vue')
    }
] 

// 创建路由实例
const router = createRouter({
    routes,
    history: createWebHashHistory('/base'), 
    // 路由模式： hash模式 - createWebHashHistory | history模式 - createWebHistory
})


const app = Vue.creatApp({})

app.use(router)

app.mount('#app')

02. router-link的变化 ​

  • 移除了tag属性——决定router-link渲染成什么类型的元素

    • 默认是渲染成<a>元素的

  • Vue3新增了v-slot属性，通过插槽的方式来决定渲染元素类型

    vue
    <template>
	<router-link v-slot="props" custom>
    	<button @click="props.navigate">
            自定义
	    </button>
    </router-link>

</template>

    • props的属性:
      • href：要跳转的链接
      • route：要跳转的目标路由对象信息
      • navigate：导航函数
      • isActive：是否匹配的状态或者说是激活状态

    • custom属性

    js
    const routes = []

const router = createRouter({
    routes,
    history: createWebHistory()
})

// 动态添加路由

// 1. 需要动态添加的路由
const newRoute = []
// 2. 使用方法动态添加
router.addRoute(newRoute)
// 3. 添加二级路由对象：第一个参数为一级路由名称，第二个参数为需要添加的二级路由对象
router.addRoute('new', newRoute)

// 1. 添加一个相同 name 的路由，会删除并替代原来的路由
router.addRoute(newRoute)

// 2. 使用 removeRoute 方法, 传入需要删除的路由 name
router.removeRoute('Home')

// 3. 通过 addRoute 的返回值回调
const removeRoute = router.addRoute(oldRoute)

08. 路由导航守卫 ​

  • 导航守卫的参数变化——两个参数：
    • to：即将前往的路由Route对象
    • from：即将离开的路由Route对象
    • 可选next参数——Vue3中不建议使用
  • 导航守卫会有一个返回值：
    • false：取消当前导航
    • 没有返回值或者返回值为undefined：进行默认导航
    • 返回一个路由地址
      • 可以是一个String类型的路径
      • 可以是一个对象，对象中包含path、query、params等路由信息

注意：

  • <script setup>中是无法使用beforeRouteEnter的
  • 需要单独在<script>中使用
