Vue3组件的多种定义方式 ​

01. defineComponent ​

  • 按照Vue2的选项式写法，通过defineComponent可以获得更好的类型提示

    vue
    <template>
	<div>Component1</div>
</template>
<script>
import { defineComponent } from 'vue';

export default defineComponent({
    name: 'Component1',
    props: {},
    emits: {},
    setup(props, context) {
        return {};
    }
})
</script>

02. render() ​

  • 不写template，直接通过render()方法返回一个Dom

    • 需要使用createVnode或者h来渲染这个Dom
    vue
    <script>
import { defineComponent, createVnode } from 'vue';

export default defineComponent({
    name: 'Component3',
    render() {
        return createVnode('div', 'Component3');
    }
})
</script>

    name: 'Component5',
    setup() {
        return () => h('div', 'Component5');

05. jsx ​

  • 使用jsx文件定义组件，其内部实现基本同上<script>标签内部

    tsx
    import { defineComponent } from 'vue';

export default defineComponent({
    props: {},
    setup(props, context) {
        return () => <div>Component11</div>
    }
})
                                
export default defineComponent((props, context) => {
    return () => <div>Component12</div>
})

06. 总结 ​

  • 大多数情况下都可以直接使用<script lang="tsx" setup>的方式来定义组件，兼容了单文件组件的优点和jsx的灵活性

    需要注意的是：函数式组件的props如果没有显示声明，则其接收的属性通context.attrs一样，包含了从父组件传下来的所有属性attributes

    • 但是可以通过解构的方式{...}来获取自己想要的属性
    • 或者标注类型的方式，来获得属性提示（虽然它还是会包含所有属性）
    vue
    <script lang="tsx" setup>
    // 函数式组件
    interface IMyComponent {}
    const MyComponent = (props: IMyComponent, context) => {
        return (
        	<div class="my-component">{ 'My Component' }</div>
        )
    }
</script>
<style scoped>
    /* 可以正常使用 scoped 作用域内的样式 */
    .my-component{
        color: red;
        color: blue;
    }
</style>

注：

  • 函数式组件是，一种定义自身没有任何状态的组件的方式
  • 因此，函数式组件内不能定义变量状态，函数式组件在状态改变时，会导致组件重新渲染，进而导致状态重置
  • 所以，尽量还是不要使用函数式组件，或者仅用于无状态组件
