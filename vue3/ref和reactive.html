ref和reactvie ​

01. ref ​

（1）基本概念 ​

  • 需要引入refAPI，来定义一个ref() 函数，这个函数可以接收一个参数值，并返回一个响应式的可改变的ref对象

    • 参数值可以是任意类型的数据

  • 返回的对象拥有一个单一属性.value来获取内部值

  • 值的访问：

    • 在setup函数内部访问ref数据需要通过.value来访问
    • 在模板中使用则不需要.value，模板中会自动对ref对象进行解包

  • 需要注意：

    • 如果data()属性与setup()函数有相同的返回值，则以setup()函数为准

（2）定义ref响应式数据 ​

  • 标注类型：使用泛型参数的方式来标注ref变量的类型

  • 代码示例：

    typescript
    // <script setup lang="ts">

// 先引入 ref
import { ref } from 'vue';

// 1. 使用 ref 定义
// -- 定义一个 number 类型的数据
const num = ref<number>(10)
// -- 定义一个 string 类型的数据
const msg = ref<string>('Vueeee')
// -- 定义一个 number 类型的数组数据
const arr1 = ref<number[]>([1, 2, 3])
const arr2 = ref<Array<number>>([1, 2, 3])
// -- 定义一个 对象
const object = ref<{ name: string; age: number }>({ name: 'Ruovan', age: 24 })

// 2. 使用，在内部需要使用 .value 访问 ref 的值
console.log(num.value)
console.log(msg.value)

// 3. 使用 ref 包装
// -- 先在外部定义一个普通数据
const common = 'Common data'
// -- 然后用 ref 函数将其转换成响应式的
// -- 新的数据与源数据不相互关联，即 newCommon 的改变 不会引起 commong 变量 的改变
const newCommon = ref<string>(common)

（3）获取元素 ​

  • 定义一个ref对象，然后绑定到元素或者组件的ref属性上，可以获取到元素或组件实例

    • 普通元素可以直接标注为HTMLElement类型
    • 自定义组件可以通过InstanceType来获取实例类型
      • 注意：这个实例只能访问到子组件expose出去的属性或方法
    html
    <template>
    <div ref="divRef"></div>
    <MyComponent ref="myComponentRef" />
</template>

<script setup lang="ts">
    import { ref, onMonted } from 'vue';
    import type { ComponentPublicInstance } from 'vue'
    import MyComponent from './MyComponent.vue';

    // 对于普通的元素，可以简单标注为 HTMLElement 类型
    const divRef = ref<HTMLElement | null>(null);

    // 对于自定义组件，可以通过 InstanceType 来获取其实例类型
    const myComponentRef = ref<InstanceType<typeof MyComponent> | null>(null)

    // 对于不关心类型的实例，可以通过 ComponentPublicInstance
    const otherRef = ref<ComponentPublicInstance | null>(null)

    // 可以在 onMounted 中拿到实例
    onMonted(() => {
        console.log(divRef.value)
        console.log(myComponentRef.value)
    })
</script>

（4）其它的 ​

  • shallowRef

    • 创建一个浅层的ref对象，使用shallowRef只会代理第一层数据，而不会对数据的进行深度代理

  • triggerRef

    • 手动触发和 shallowRef 相关联的副作用

  • customRef

    • 创建一个自定义的ref，并对其依赖项跟踪和更新触发进行显示控制

02. reactive ​

（1）基本概念 ​

  • 需要引入一个reactiveAPI，来定义一个reactive() 函数，这个函数需要接收一个引用类型的数据，并返回一个响应式代理器对象

    • 如果不是引用类型，就会抛出错误
    • reactive会将每一层的数据都解析成为proxy对象，即每一层的数据都是响应式的

  • 值的访问：

    • 直接使用普通对象的访问形式

  • 需要注意：

    • reactive定义的变量不能直接赋值，否则会丢失响应式
    • 同时，也不建议将新值用reactive包裹赋值给原变量，同样会丢失响应式

（2）定义reactive响应式数据 ​

  • 标注类型

    • 推荐使用接口的方式来标注reactvie变量的类型
    • 不推荐使用泛型参数的方式来标注类型

  • 代码示例：

    typescript
    // <script setup lang="ts">

// 引入 reactive
import { reactive } from 'vue';

// 定义一个接口
interface IObj {
    name: string
    age: number
}

// 定义响应式数据对象
// -- 泛型参数的形式不推荐
const state = reactive<IObj>({
    name: 'haha',
    age: 18
})
// -- 推荐使用这种方式来标注 reactvie 变量的类型
const state: IObj = reactive({
    name: 'haha',
    age: 18
})

// 可以直接访问
console.log(state.name)
console.log(state.age)

（3）其它的 ​

  • shallowReactive
    • 可以创建一个响应式代理proxy
    • 但不执行嵌套对象的深度响应式转换
    • 即只有第一层是响应式的对象，但是嵌套的深层对象还是原生对象

建议使用ref来定义基本类型的数据，使用reactive定义复杂类型的数据

03. readonly ​

  • 基本介绍

    • readonly()会返回原生对象的只读代理，不允许对其进行修改
    • readonly()可以接收三种类型的参数
      • 普通对象类型
      • ref对象类型
      • reactive对象类型
    • 应用：在传递给其他组件数据时，往往希望其他组件使用我们传递的内容，但是不允许它们修改时，就可以使用readonly了

  • shallowReadonly

    • 创建一个 proxy，使其自身的 property 为只读
    • 但不执行嵌套对象的深度只读转换
    • 即，只有第一层是只读的，嵌套的深层对象还是可以读写的

04. 工具函数 ​

（1）toRef ​

  • 基本介绍

    • 使用toRef可以将一个普通数据变成响应式数据ref
      • 且新数据与原数据是关联的（这一点与ref不同），但数据变化不会触发模板（页面）上的数据更新
      • 接收两个参数，第一个参数是对象，第二个参数是对象的某个属性
    • 另一个作用是将 reactive() 创建出来的响应式对象的某一个属性，转换为ref对象

  • 代码示例

    js
    import { defineComponent, reactive, ref, toRef } from 'vue';

export default defineComponent({
    setup(props, context){
        // 定义一个引用类型的普通数据
        let obj = {
            a: 1,
            b: 2
        }
        // 将 obj.a 变成响应式
        let obj_a = toRef(obj, 'a')
        // 修改响应式数据可以影响原数据，但不会触发模板（页面）上的数据更新
        obj_a.value = 10
        console.log(obj_a) // log: 10
        console.log(obj.a) // log: 10


        const state = reactive({
            name: 'Ruovan',
            age: 24,
        })
        const nameRef = toRef(state, name) 

        return { 
            obj_a
        }
    }
})

（2）toRefs ​

  • 基本介绍

    • 使用toRefs可以一次性转换对象的多个属性为响应式的ref类型的属性
      • 需要注意的是，转换之后的数据访问仍然需要通过.value，因为已经是ref类型的属性了
      • 数据的改变还是不会触发页面的更新
    • 另一个作用是将 reactive() 创建出来的响应式对象，转换为多个普通的ref对象
      • 即，将响应式的reactive对象拆分成多个ref()类型的响应式数据
    • 如果使用ES6的解构语法进行解构，那么内部所有属性都会失去响应式

  • 对比：

    • toRef针对单个属性
    • toRefs可以转换多个属性

  • 代码示例

    js
    import { defineComponent, reactive, toRefs } from 'vue';

export default defineComponent({
    setup(props, context){
        // 定义一个普通数据
        let obj = {
            a: 1,
            b: 2
        }
        let newObj = toRefs(obj)
        newObj.a.value = 10
        newObj.b.value = 20

        console.log(newObj)
        console.log(obj)

        // ----
        let state = reactive({
            name: 'haha',
            age: 18
        })

        return { 
            ...toRefs(state) // 相当于解构，在模板中直接使用state的属性 —— name、age
        }
    }
})

（3）unref ​

  • 获取一个ref引用中的value
    • 如果参数不是一个ref对象，那么返回参数本身
    • 否则，返回参数代理的值

（4）toRaw ​

  • 返回reactvie或readonly代理的原始对象，谨慎使用

05. 检测响应式 ​

（1）isRef ​

  • 检查对象是否是由ref创建的，返回值为boolean类型

（2）isReactive ​

  • 检查对象是否是由 reactive 创建的响应式代理，返回值为boolean类型
  • 如果该代理是 readonly 创建的，但包裹了由 reactive 创建的另一个代理，它也会返回 true

（3）isReadonly ​

  • 检查对象是否是由 readonly 创建的只读代理

（4）isProxy ​

  • 检查对象是否是由 reactive 或 readonly 创建的 proxy

10. 问题收集 ​

（1）数组更新赋值问题（3种方法） ​

  • 使用reactvie方法直接定义数组，可以使用数组方法的形式来更新数组

    typescript
    const arr = reactive([])
arr.push(...[1, 2, 3])

  • 使用reactive方法间接定义数组，即将数组作为对象的一个属性，这样可以通过直接赋值的形式更新数组

    typescript
    const state = reactive({
  arr: [],
})
state.arr = [1, 2, 3]

    typescript

（2）解除响应式数据响应的问题 ​

  • 使用ref、reactive定义的响应式数据
  • 使用unref、toRaw、markRaw都只能解除第一层响应，嵌套的对象的响应式无法解除
  • 可以使用深浅拷贝的形式来解除响应式
