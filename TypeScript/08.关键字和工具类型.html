01. extends关键字 ​

  • 语法：A extends B

  • 作用：检验A是否拥有B中的属性

    typescript
    interface Length {
  length: number;
}

const func = <T extends Length>(data: T): number => {
  return data.length;
};

func([1, 2]); // 使用时，会检验传入的变量类型 是否有 length 这个属性
func(12); // 如果没有 length 属性， 就会报错

03. keyof关键字 ​

  • 语法：keyof A

  • 作用：获取类型A的所有的公共键key，其返回类型是联合类型，可以检查对象上的键是否存在

    typescript
    interface Person {
  name: string;
  age: number;
}

type NewPerson = keyof Person; // "name" | "age
type NewPersonArr = keyof Person[]; // "length" | "toString" | "pop" | "push" | "concat" | "join

// 泛型中应用
const getInfo = <T, K extends keyof T>(data: T, key: K): T[K] => {
  return data[key];
};

const user = {
  name: 'Ruovan',
  age: 24,
};

getInfo(user, 'name'); // 返回 'Ruovan'
getInfo(user, 'address'); // 报错， user 里没有 address 这个属性

06. infer（似懂非懂） ​

  • 语法：

  • 作用：使用为条件语句，可以用infer声明一个类型变量并且对它进行使用

    • 可以推断一个类型变量，高效的对类型进行模式匹配

    • infer只能在extends的右边使用
    typescript

type Person = Info<{ name: string; age: number }>; // Person: string | number

type Props = Info<number>; // Props: never

07. Partial和Required和Readonly和Record ​

（1）概念 ​

  • Partial<T>：将类型T中的所有属性变为可选属性
  • Required<T>：将类型T中的所有属性变为必选属性
  • Readonly<T>：将类型T中的所有属性变为只读属性
  • Record<T, U>：将类型 T 中所有的属性都转化为指定类型U
    • 需要注意的是：类型T必须是联合类型

（2）示例 ​

  • 示例：

    typescript
    interface Person {
  name: string;
  age: number;
  sex?: string;
}

// 1. Partial
type PartialPerson = Partial<Person>;
// ===>
type PartialPerson = {
  name?: string;
  age?: number;
  sex?: string;
};

// 2. Required
type RequiredPerson = Required<Person>;
// ===>
type RequiredPerson = {
  name: string;
  age: number;
  sex: string;
};

// 3. Readonly
type ReadonlyPerson = Readonly<Person>;
// ===>
type ReadonlyPerson = {
  readonly name: string;
  readonly age: number;
  readonly sex: string;
};

// 4. Record
type Mine = 'Ruovan' | 'Composition';
type RecordPerson = Record<Mine, Person>;
// ===>
type RecordPerson = {
  Ruovan: Person;
  Composition: Person;
};

（3）原理 ​

  • 原理

    typescript
    // 通过映射类型遍历 T 上所有的属性，然后对每个属性进行设置

// 添加 ? 可选修饰符
type Partial<T> = {
  [P in keyof T]?: T[P];
};

// -? 表示去除可选修饰符
type Required<T> = {
  [P in keyof T]-?: T[P];
};

// 添加 readonly 修饰符
type Readonly<T> = {
  readonly [P in keyof T]: T[P];
};

// 转为为指定类型 U
type Record<T extends keyof any, U> = {
  [P in T]: U;
};

11. Pick和Omit ​

（1）概念 ​

  • Pick<T, U>：将T类型中的子属性U挑选出来，获得挑选出来的当前类型的子类型
    • U = K extends keyof T>
  • Omit<T, U>：将T类型中的子属性U剔除出去，获得属性剔除后的新类型
    • Omit返回的是新的类型，原理上是在 Exclude之上进行的，Exclude是根据自类型返回的

（2）实例 ​

  • 示例

    typescript
    interface Person {
  name: string;
  age: number;
  sex: string;
}

// 1. Pick
type PickPerson = Pick<Person, 'sex'>;
// ===>
type PickPerson = {
  sex: string;
};

// 2. Omit
type OmitPerson = Omit<Person, 'sex'>;
// ===>
type OmitPerson = {
  name: string;
  age: number;
};

12. Exclude和Extract ​

（1）概念 ​

  • Exclude<T, U>：将联合类型T中的U类型剔除出去

  • Extract<T, U>：将联合类型T中的U类型提取出来

    • T类型必须是联合类型
    • 如果U类型有的属性T中没有，那么会返回never

（2）示例 ​

  • 实例

    typescript
    interface Person {
  name: string;
  age: number;
  sex: string;
}

// 1. Exclude
type ExcludePerson = Exclude<keyof Person, 'sex'>;
// ===>
type ExcludePerson = 'name' | 'age';

// 2. Extract
type ExtraPerson = Extra<keyof Person, 'sex'>;
// ===>
type ExtraPerson = 'sex';

（3）原理 ​

  • 原理

    typescript
    // 通过条件类型来判断，如果类型 T 可被分配给类型 U ，则不返回类型 T，否则返回此类型 T
// -- Exclude 类型相当于取出两个联合类型的 差集
type Exclude<T, U> = T extends U ? never : T;

// 与 Exclude 相反
// Extract 类型相当于取出两个联合类型的 交集
type Extract<T, U> = T extends U ? T : never;

16. ReturnType和Parameters ​

（1）概念 ​

  • ReturnType<T>：获取函数类型T的返回类型

  • Parameters<T>：获取函数类型T的参数类型

    • 参数类型是一个数组，如果没有参数，则为空数组

（2）示例 ​

  • 示例

    typescript
    type Func = (a: string, b: number) => string;

// 1. ReturnType
type ReturnTypeFunc = ReturnType<Func>; // string

// 2. Parameters
type ParametersFunc = Parameters<Func>; //  [a: string, b: number]
