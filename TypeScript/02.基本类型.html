基本类型使用 ​

00. 前言 ​

  • 在变量名之后添加类型注解

    • 类型注解：一种轻量级的为函数或变量添加约束的方式

  • TypeScript提供了静态的代码分析，它可以分析代码结构和提供的类型注解

02. 数组和元组 ​

（1）数组 ​

  • 数组的定义方式有两种：

    • 第一种：在元素类型后加上Type[]

      typescript
      1

    // 类型 后接 [] let arr: numer[] = [1, 2, 3]

    • 第二种：使用数组泛型Array<元素类型>

      typescript

    • 类型可以通过|分开，表示可以是多种类型的元素

（2）元组 ​

  • 数组一般由同种类型的值组成
    • 但有时我们需要在单个变量中存储不同类型的值
    • 这时候我们就可以使用元组
  • 元组
    • 元组类型允许表示一个已知元素数量和类型的数组
      • 同时也是限制数组元素的数量和类型
    • 元组各元素的类型不必相同
    • 元组赋值时，必须提供每个属性的值
    • 元组的使用方式同数组
typescript
// 定义一个名为 tuple 的元组，需要按 [正确的类型顺序]，依次初始化 tuple 元组

// 可以通过 ? 来标识元素为可选
let tuple2: [string, number, boolean?] = ['tuple2', 123]

// 可以使用 readonly 来表示只读，不能修改元组中的元素，否则报错
let tuple3: readonly [string, number] = ['tuple3', 123]

03. 枚举Enum ​

（1）概念 ​

  • 介绍：
    • 枚举：可以用来定义一些带名字的常量，将一组可能出现的值，一一枚举列出来
    • 使用Enum来标识一个枚举
  • 应用：
    • 用某几个数值去代表某种状态
    • 使用枚举定义一些带名字的常量
  • 命名方式：
    • 枚举名：首字母大写
    • 枚举值：属性全大写

（2）数字枚举 ​

  • 枚举的所有成员默认为Number类型

    • 默认从0 开始编号，后面的元素依次递增

  • 直接使用：

    typescript
    // 定义一个枚举类型
enum Demo {
    A,
    B    
}

// 使用
Demo.A // 0
Demo.B // 1

（6）遍历枚举 ​

  • 枚举可以通过两种方式来进行遍历

    • 第一种：使用for...in来进行遍历，但无法通过for...of来遍历
    • 第二种：通过Object.keys(enum)和Object.values(enum)来遍历
    typescript
    // 枚举的遍历
enum fruitEnum {
  APPLE = 'APPLE_VALUE',
  ORANGE = 'ORANGE_VALUE',
  BANANA = 'BANANA_VALUE'
}

// log: APPLE ORANGE BANANA
for (const key in fruitEnum) {
  console.log(key)
}

// 报错: fruitEnum is not iterable
for (const value of fruitEnum) {
  console.log(key)
}

// log: ['APPLE', 'ORANGE', 'BANANA']
console.log(Object.keys(fruitEnum))

// log: ['APPLE_VALUE', 'ORANGE_VALUE', 'BANANA_VALUE']
console.log(Object.values(fruitEnum))

// 如果不需要其中给的数字，可以将数字过滤掉
// log: ['A', 'B', 'C', 'D']

// 但对于values来说就不友好了
// log: ['A', 'B', 'C', 'W']

// 所以，可以借助 keys 来完成
// log: [0, 1, 2, 'W']

  Object.keys(testEnum)
    .filter(key => Number.isNaN(Number(key)))
    .map(key => testEnum[key])
)

05. any和unknown ​

（1）any ​

  • any类型可以表示任何类型，这让 any 类型成为了TypeScript的顶级类型（也被称作全局超级类型）

    • 它可以被赋值为任何类型，甚至使用任何方法
    • 可以使用 any 类型来标记在编程阶段还不清楚类型的变量
    • 可以对 any 类型的值执行任何操作，而无需事先执行任何形式的检查
    typescript
    // any类型
let notSure:any = 10
notSure = '10'
notSure = true
    1
    2
    3
    4
    • 使用any类型后，就失去了TypeScript的类型限制，能不使用any就不使用any

（2）unknown ​

  • unknown类型表示未知类型、不确定的类型，是另一种顶级类型

    • 它可以被赋值为任何类型，但不能使用任何方法，因为它是未知的
    • unknown 类型只能被赋值给 any 类型和 unknown 类型本身
    typescript
    // unknown 类型
let value: unknown;
value = true
value = 42

// 无法赋值为其它类型
let value1: number = value // Error
let value2: string = value // Error

// 只能赋值给 any | unknown
let value3: any = value
let value4: unknown = value

06. void和never ​

（1）void ​

  • void类型表示没有任何类型——空类型

    • 通常，一个函数没有返回值时，其返回值类型是void
    • 在声明函数返回值类型时，在方法名后面添加类型注解
    typescript
    // 函数： 表示函数的返回值为空（null || undefined）
function func1(): void {
    console.log('-------')
}

let unusable2: void = undefined

（2）never ​

  • never类型表示永不存在的值的类型

    • 例如，never 类型是那些总是会抛出异常、或者根本就不会有返回值的函数表达式、或者箭头函数表达式，的返回值类型

  • 使用 never 可以避免出现新增了联合类型没有对应的实现，目的就是写出类型绝对安全的代码

    typescript
    // 返回never的函数必须存在无法达到的终点
function error(message: string): never {
    throw new Error(message);
}

function infiniteLoop(): never {
    while (true) {}
}
